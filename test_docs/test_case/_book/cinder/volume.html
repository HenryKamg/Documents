01060202 测试创建"eqlx"类型的卷
  • UI:
    1. 登录到 dashboard；

    3. 执行测试命令 rally task start create-volumes.yaml。
  • UI:
    • 卷创建成功
    • 卷的状态变化为 Creating -> Available
  • CLI:
    • Rally 测试成功
 执行 5 次，每次并行 2 个测试 Rally:

01060204 创建一个大于配额的卷
  • 前提：
    • 为了方便测试，创建一个配额很小的 Project，其中，卷的大小配额为 5 G
  • UI:
    1. 以所创建的 Project 中的用户登录到 dashboard；
    2. 点击左侧导航栏的 【Project】，展开选项卡；
    3. 点击 【Compute】 子选项卡，展开；点击 【Volumes】 选项；
    4. 点击 【Create Volume】 按钮；
    5. 在弹出的对话框中，输入 Volume Name，设置卷的大小 Size 为 10 G；
    6. 其他配置保持默认，点击窗口下方的 【Create Volume】 按钮。
  • CLI:
    1. 登录到 controller 节点，使用所创建的测试 Project 执行操作；
    2. 执行命令 cinder create 10。
  • UI:
    • 不能创建卷
    • 在窗口上方提示错误信息为："A volume of 10GB cannot be created as you only have 5GB of your quota available."
  • CLI:
    • 无法创建卷
    • 提示的错误信息为："ERROR: Request Entity Too Large (HTTP 413)"
 Rally 无法指定使用的项目，因此考虑直接使用命令行 None

01060216 将已被附加的卷(即 "In-Use" 状态)卷上传到镜像，使用 QCOW2 格式
  • UI:
    1. 登录到 dashboard；
    2. 点击左侧导航栏的 【Project】，展开选项卡；
    3. 点击 【Compute】 子选项卡，展开；
    4. 点击 【Volumes】 选项；
    5. 选择一个卷，点击该卷的 【Actions】 下拉按钮，点击 【Upload to Image】；
    6. 在弹出的 【Upload Volume to Image】 窗口中，填写 Image Name，选择 Disk Format 为 QCOW2，勾选 Force 选项；
    7. 点击 【Upload】 按钮
  • CLI:
    1. 登录到 controller 节点；
    2. 执行命令 cinder upload-to-image \ --container-format bare --disk-format qcow2 \，其中所选择的卷已经附加给某个示例。
  • UI:
    • 卷创建成功，卷的状态变化为 Creating -> Available
    • 卷上传到镜像成功，卷的状态变化为 Uploading -> Available
    • 点击 【Images】 选项，看到列表中列出了所上传的镜像，状态为 Active
  • CLI:
    • 命令执行失败，提示信息为："ERROR: Bad Request (HTTP 400)"
    • 添加 --force True 参数重新执行命令，命令执行成功，将卷上传为镜像
 Rally 修改 force: true 参数不生效 None

01060220 从 eqlx 快照中创建 eqlx 卷的功能
  • UI:
    1. 登录到 dashboard；
    2. 点击左侧导航栏的 【Project】，展开选项卡；
    3. 点击 【Compute】 子选项卡，展开；
    4. 点击 【Volumes】 选项；
    5. 在右侧点击 【Volume Snapshot】 子标签，选择一个基于 eqlx 卷创建的快照，点击 【Actions】 中的 【Create Volume】；
    6. 在弹出的 【Create Volume】 窗口中，填写 Volume Name，选择 Type 为 "eqlx"，其他配置保持默认；
    7. 点击窗口下方的 【Create Volume】 按钮。
  • CLI:
    1. 登录到 Controller 节点；
    2. 执行命令 cinder create --snapshot-id \ --volume-type eqlx --display-name \ \。
    		•
    • UI:
      • 卷创建成功，卷的状态变化为 Creating -> Available
    • CLI:
      • 命令执行成功，输出所创建的卷的信息，状态为 Creating
      • 使用 cinder show \ 查看该卷，可以看到卷的状态变为 Available
    		 None
