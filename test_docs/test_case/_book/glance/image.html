01080102 测试本地上传 QCOW2 镜像的功能
  • UI:
    1. 登录到 dashboard；
    2. 点击左侧导航栏的 【Project】，展开选项卡；
    3. 点击 【Compute】 子选项卡，展开；
    4. 点击 【Images】选项，点击 【Create Imange】 按钮；
    5. 在弹出的 【Create An Image】 窗口中，填写 Name，选择 Image Source 为 Image File，点击选择需要上传的本地镜像文件；
    6. 选择 Format 为 QCOW2，其他设置保持默认；
    7. 点击 【Create Image】 按钮，上传镜像
  • CLI:
    1. 登录到 rally 测试服务器；
    2. 使用 rally 测试文件 create-and-list-image_qcow2_local.yaml；
    3. 执行测试命令 rally task start create-and-list-image_qcow2_local.yaml 。
  • UI:
    • 镜像上传成功
    • 镜像的状态变化为 Queued -> Spaning -> Active
    • 列表中显示新建的镜像的信息
  • CLI:
    • Rally 测试成功
 执行 10 次，每次执行 1 个测试 Rally:
create-and-list-image_qcow2_local.yaml

01080107 测试编辑镜像的功能
  • UI:
    1. 登录到 dashboard；
    2. 点击左侧导航栏的 【Project】，展开选项卡；
    3. 点击 【Compute】 子选项卡，展开；
    4. 点击 【Images】选项，在右侧选择一个类型为 "Image" 的镜像，点击 【Actions】 中的 【Edit】；
    5. 在弹出的 【Update Image】 窗口中，可以修改 Format, Minimum Disk, Minimum RAM，勾选 Public 和 Protected 使镜像成为共用的或受保护的
    6. 修改完成后，点击窗口下方的 【Update Image】 按钮。
  • CLI:
    1. 登录到 Controller 节点；
    2. 执行命令 glance image-update \，中间可以添加需要修改的参数。
  • UI:
    • 镜像修改成功，有 Success 的信息提示
  • CLI:
    • 命令执行成功，镜像被修改
 None

01080111 测试上传的 QCOW2 镜像能否正常使用
  • UI:
    1. 登录到 dashboard；
    2. 点击左侧导航栏的 【Project】，展开选项卡；
    3. 点击 【Compute】 子选项卡，展开；
    4. 点击 【Images】 选项，选择一个新创建的 QCOW2 镜像，点击 【Actions】 中的 【Launch】；
    5. 在弹出的 【Launch Instance】 窗口中的 Details 标签中，输入 Instance Name，选择 Flavor 为 m1.tiny，Instance Count (实例数量) 为 1，其他配置保持默认；
    6. 点击 Networking 标签，选择为实例分配的网络；
    7. 其他标签的配置保持默认，点击 【Launch】 按钮。
  • CLI:
    1. 登录到 rally 测试服务器；
    2. 使用 rally 测试文件 create-image-and-boot-instances_qcow2.yaml；
    3. 执行测试命令 rally task start create-image-and-boot-instances_qcow2.yaml。
  • UI:
    • 镜像创建成功，镜像的状态变化为 Queued -> Spaning -> Active
    • 实例创建成功，实例的状态变化为 Build -> Active，任务变化为 Spawning -> None
    • 选择所创建的实例，点击 【Actions】 中的 【Console】，打开实例控制台，所创建的实例的操作系统与所上传的镜像相符
  • CLI:
    • Rally 测试成功
  • 执行 10 次，每次执行 1 个测试
  • 每次创建 2 个 tenant，每个 tenant 包含 2 个 user
  • 每次启动 2 台示例
 Rally:
create-image-and-boot-instances_qcow2.yaml
