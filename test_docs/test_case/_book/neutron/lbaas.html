01010401 创建资源池
  • UI:
    1. 登录dashboard，点击【Project】 选项，选择【Network】子选项；
    2. 点击 【Networks】 选项，点击【负载均衡】子选项，进入负载均衡配置界面；
    3. 点击资源池按钮创建一个新的资源池，点击资源池，弹出新增资源池对话框；
    4. 名称填写“lb-pool”，提供者使用默认着“haproxy”，子网选择一个你需要去负载均衡的子网网段，协议“TCP”，负载均衡方式“ROUND——ROBIN”，点击添加按钮资源池创建成功。
  • CLI:
    1. 登录到controller节点；
    2. 执行命令：neutron lb-pool-create --lb-method ROUND_ROBIN --name pool2 --protocol TCP --subnet-id SUBNET-ID
  • UI:
    1. 资源池创建成功。
  • CLI:
    1. 资源池创建成功。
 协议类型可以根据需求选择
01010402 创建VIP
  • UI:
    1. 登录dashboard，点击【Project】 选项，选择【Network】子选项；
    2. 点击 【Networks】 选项，点击【负载均衡】子选项，进入负载均衡配置界面；
    3. 点击资源池按钮，选择需要添加VIP的资源池，点击下拉菜单点击添加VIP；
    4. 名称填写“lb-vip”，选择负载均衡中子网中一个可用IP填写到“从选定的子网中指定一个可用的IP地址”，协议端口“22”，点击添加按钮，VIP创建成功。
  • CLI:
    1. 登录到controller节点；
    2. 执行命令： neutron lb-vip-create --name vip2 --protocol-port 22 --protocol TCP --subnet-id SUBNET-ID POOL-ID
  • UI:
    1. 资源池VIP创建成功。
  • CLI:
    1. 资源池VIP创建成功。
 端口号根据协议类型以及需求选择
01010403 创建资源池成员
  • UI:
    1. 登录dashboard，点击【Project】 选项，选择【Network】子选项；
    2. 点击 【Networks】 选项，点击【负载均衡】子选项，进入负载均衡配置界面；

    2. 执行命令：neutron lb-member-create --address 192.168.202.6 --protocol-port 22 POOL-ID

  • CLI:
    1. 资源池成员创建成功。
 端口号根据协议类型以及需求选择，获取主机IP可以执行命令nova list获取到
01010404 创建监控
  • UI:
    1. 登录dashboard，点击【Project】 选项，选择【Network】子选项；
    2. 点击 【Networks】 选项，点击【负载均衡】子选项，进入负载均衡配置界面；
    3. 点击监控按钮，创建监控，点击新增监控弹出新增监控对话框；
    4. 类型选择“PING”，延迟“1”，超时“1”，最大重试次数“2”，点击添加按钮，监控创建成功；
    5. 将监控与资源池绑定，点击资源池按钮，选定需要绑定监控的资源池，点击下拉菜单；
    6. 选择关联监控，弹出关联监控对话框，选择需要绑定的监控，点击关联。
  • CLI:
    1. 登录到controller节点；
    2. 执行命令创建监控：neutron lb-healthmonitor-create --delay 3 --type PING --max-retries 3 --timeout 3
    3. 执行命令将监控与资源池绑定：neutron lb-healthmonitor-associate LB-HEALTHMONITOR-ID POOL-ID
  • UI:
    1. 监控创建成功。
  • CLI:
    1. 监控创建成功。
 延迟、时延、最大重试次数根据实际可行情况填写
01010405 修改资源池
  • UI:

    3. 执行命令修改负载均衡方式：neutron lb-pool-update POOLNAME --lb_method SOURCE_IP。
  • UI:
    1. 资源池修改成功。
  • CLI:
    1. 资源池修改成功。
